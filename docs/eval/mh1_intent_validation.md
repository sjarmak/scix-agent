# MH-1 Intent Backfill — Hand Validation Sample

This report scaffolds hand validation of the SciBERT-SciCite citation intent classifier output (PRD `docs/prd/scix_deep_search_v1.md` §MH-1).

## Methodology

- Sample size: 500 (stratified by predicted class — equal share per class, remainder to the lowest-indexed class).
- Random seed: fixed (default 42) so the sample is reproducible across runs.
- Source: rows from `citation_contexts` after the SciBERT-SciCite backfill completes. NULL `intent` rows are excluded.
- Confidence: raw classifier score from `transformers.pipeline('text-classification', ...)`. Calibration is out-of-distribution (model trained on CS/biomed); treat as a relative ordering only.
- Hand-label workflow: domain expert fills the `manual_label_placeholder` column with one of `background | method | result_comparison | unsure`. Disagreements feed the v1.1 escalation gate.

## Throughput / Operator Notes

- Expected throughput on RTX 5090: ~12 rec/s (SciBERT inference; same envelope as the INDUS embedding pipeline per `CLAUDE.md`).
- Full-corpus run (823K rows): ~1 GPU-day. Operator-only — the work unit ships the script + smoke-test mode (`--smoke-test`, 100 rows), not the full GPU pass.
- Per-row overhead: ~1/5000 s ≈ 0.2 ms wall clock under saturation. Wrap heavyweight runs in `scix-batch` per `CLAUDE.md` memory-isolation guidance.
- Corpus coverage at sample time: 823,000 classified rows.
