## Scope (when activated)

### Is in scope

- Production-side `CREATE EXTENSION vectorscale;` on the `scix`
  database (one-time extension install, version-pinned).
- Build a new pgvectorscale StreamingDiskANN index on
  `paper_embeddings WHERE model_name='indus'` using the exact
  parameter set from the winning variant identified in the decision
  doc.
- Dual-read flag in `src/scix/search.py` so the dense retrieval path
  can target either the HNSW index or the new DiskANN index; traffic
  ramp from 5% → 25% → 100% over a one-week dual-read window.
- Rollback script that flips the flag back to HNSW (config-level
  rollback, no rebuild) and, as a second step, drops the DiskANN
  index cleanly if we abandon the migration.
- Update `src/scix/search.py` (and any MCP tools that bypass it) to
  query the new index once cutover is complete.
- Monitoring hooks: live nDCG@10 on the 50-query eval, p95 latency,
  extension-level errors in Postgres logs / `pg_stat_statements`.
- Post-cutover: retain the HNSW index for one week as a hot fallback,
  then drop it in a follow-up window.

### Is NOT in scope

- Re-embedding (the 32M INDUS embeddings already exist and are not
  regenerated by this migration).
- Dual-model embedding work — owned by
  `docs/prd/prd_dual_model_embedding_rerank.md`.
- Chunk-level embeddings — owned by
  `docs/prd/prd_body_chunk_embeddings.md`.
- Community detection refinement / other retrieval work — owned by
  `docs/prd/prd_community_detection_v2.md`.
- Any embedding-model change or additional ANN library beyond
  pgvector / pgvectorscale.
- Any BM25 / full-text search path changes.

## Trigger Condition

The decision doc's **Go Path** section must show:

- C1 (unfiltered nDCG@10 within 1% of HNSW) PASS
- C2 (filtered p95 at least 30% lower than HNSW) PASS
- C3 (index size <= 1.5x HNSW) PASS

all cleared by a **single** pgvectorscale variant (V1, V2, or V3).
Until that single variant is identified and recorded in the decision
doc's "Selected variant" field, this PRD remains a stub and no
migration SQL is authored.
